# План: Деплой бэкенда в Yandex Cloud Serverless Containers (v2)

## Контекст

Бэкенд: Express 5, JSON-файловое хранилище (products.json + orders.json), S3 для видео уже настроен.
Проблема: Serverless Containers не сохраняют файловую систему между запусками.

## Решение проблемы с данными

**Вариант: products в образе, orders в памяти.**

- `products.json` — начальные данные **вшиваются в Docker-образ** (read-only, не меняются часто)
- `orders.json` — хранятся **только в памяти** (заказы живут 10 минут и истекают, при рестарте контейнера теряются — это ок для тестов)
- Видео — уже загружаются напрямую в Yandex S3 через presigned URL

## Исправления (4 фатальных бага v1)

### 1. Dockerfile — порт 8080 и ENV

**Проблема v1:** Порт 5000 в Dockerfile vs 8080 в CLI. HEALTHCHECK стучался на неправильный порт. Yandex Cloud SC требует явный ENV PORT.

**Замена всего Dockerfile:**

```dockerfile
# ─── Stage 1: Build ──────────────────────────────────────────────────────────
FROM node:20-slim AS builder
WORKDIR /app
COPY server/package*.json ./
RUN npm install --production
COPY server/ .

# ─── Stage 2: Runtime ────────────────────────────────────────────────────────
FROM node:20-slim
RUN apt-get update && apt-get install -y --no-install-recommends curl && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY --from=builder /app .

# Yandex Cloud Serverless Containers требует слушать порт из переменной PORT.
# Стандарт YC SC — 8080.
ENV PORT=8080
EXPOSE 8080

HEALTHCHECK --interval=30s --timeout=5s \
  CMD curl -f http://localhost:8080/health || exit 1

CMD ["node", "server.js"]
```

### 2. server.js — isServerless, защита записи, listen на 0.0.0.0

**Проблема v1:** `saveOrders()` пытается писать файл в read-only FS серверлесса → EROFS и краш. `app.listen(PORT)` слушает только localhost → YC SC не может достучаться.

**Замена блока в `server/server.js`:**

Заменить строку 16 (`const PORT = process.env.PORT || 5000;`) на:

```js
const PORT = process.env.PORT || 8080;
const isServerless = process.env.NODE_ENV === "production" && !process.env.DATA_DIR;
```

Заменить整个 `app.listen` блок (строки 537-541) на:

```js
// ─── Healthcheck ─────────────────────────────────────────────────────────────

app.get("/health", (req, res) => {
  res.status(200).json({ status: "ok" });
});

// ─── Start ───────────────────────────────────────────────────────────────────

app.listen(PORT, "0.0.0.0", () => {
  console.log(`\n  MARX SHOP API running on http://0.0.0.0:${PORT}${isServerless ? " (serverless)" : ""}\n`);
});
```

### 3. storage.js — защита saveOrders от read-only FS

**Проблема v1:** `saveOrders()` делает `fs.writeFileSync()` — в serverless FS только для чтения, падает с EROFS.

**Замена функции `saveOrders` в `server/lib/storage.js` (строки 22-25):**

```js
let inMemoryOrders = [];

function saveOrders(orders) {
  if (process.env.NODE_ENV === "production") {
    // В серверлессе FS read-only — храним заказы только в памяти.
    // При рестарте контейнера заказы теряются (ок для тестов).
    inMemoryOrders = orders;
    return;
  }
  ensureDataDir();
  fs.writeFileSync(ORDERS_FILE, JSON.stringify(orders, null, 2), "utf-8");
}
```

Также заменить `loadOrders` (строки 12-20):

```js
function loadOrders() {
  if (process.env.NODE_ENV === "production") {
    return inMemoryOrders;
  }
  ensureDataDir();
  try {
    if (!fs.existsSync(ORDERS_FILE)) return [];
    return JSON.parse(fs.readFileSync(ORDERS_FILE, "utf-8"));
  } catch {
    return [];
  }
}
```

### 4. Шаг 2 — добавить роль storage.editor для SA

**Проблема v1:** SA не имеет прав на Object Storage. Presigned URL для загрузки видео не сгенерируется → 403.

**Замена Шага 2 в инструкции по деплою:**

```bash
# Создать Service Account
SA_ID=$(yc iam service-account create --name marxshop-deployer --format json | jq -r '.id')
FOLDER_ID=$(yc resource-manager folder get-by-name --name <имя-folder> --format json | jq -r '.id')

# Права: push в Container Registry
yc resource-manager folder add-access-binding \
  $FOLDER_ID --member serviceAccount:$SA_ID \
  --role container-registry.images.pusher

# Права: создавать и управлять Serverless Containers
yc resource-manager folder add-access-binding \
  $FOLDER_ID --member serviceAccount:$SA_ID \
  --role serverless.containers.user

# Права: доступ к Object Storage (нужно для presigned URL загрузки видео)
yc resource-manager folder add-access-binding \
  $FOLDER_ID --member serviceAccount:$SA_ID \
  --role storage.editor
```

### 5. Шаг 5/6 — привязка SA при деплое

**Проблема v1:** Контейнер запускается без SA → не может обращаться к S3 → загрузка видео падает.

**Замена команды деплоя (Шаг 6):**

```bash
# Деплой ревизии с привязкой SA (без этого S3 не работает)
yc serverless container revision deploy \
  --container-name marxshop-api \
  --image cr.yandex/<registry-id>/marxshop-backend:v1 \
  --memory 512MB \
  --concurrency 1 \
  --execution-timeout 60s \
  --service-account-id $SA_ID \
  --env NODE_ENV=production \
  --env PORT=8080 \
  --env YC_BUCKET=marx-shop-videos \
  --env YC_REGION=ru-central1 \
  --env YC_ACCESS_KEY=<access-key> \
  --env YC_SECRET_KEY=<secret-key> \
  --env YC_PUBLIC_URL=https://storage.yandexcloud.net/marx-shop-videos

# Получить публичный URL
yc serverless container list
# URL: https://<container-id>.containers.yandexcloud.net
```
