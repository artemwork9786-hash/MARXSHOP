# План: Исправление вертикального ползунка громкости

## Проблема
Ползунок громкости не таскается пальцем. Две корневые причины:
1. `handleVolumeChange` передаёт значение 0-100 в `video.volume`, а нужно 0-1
2. `transform: rotate(-90deg)` на `<input type="range">` ломает touch-координаты

## Решение: Замена на pointer events вместо rotated input

### Файл: `src/components/AccountCard.jsx`

### Изменение 1: Исправить `handleVolumeChange`
```js
// БЫЛО:
v.volume = val;

// СТАЛО:
v.volume = val / 100;
```

### Изменение 2: Переписать контейнер громкости на pointer events

Заменить блок `{showVolume && (...)}` на реализацию с `onPointerDown`/`onPointerMove`/`onPointerUp`:

- Контейнер `div` с `ref={volumeRef}`, `relative h-24 w-8`
- Обработчик `calcVolumeFromY(clientY)` — вычисляет процент по Y-координате относительно контейнера
- `onPointerDown` → начинаем drag, вычисляем громкость
- `onPointerMove` → обновляем громкость (только если button === 0, т.е. левая кнопка мыши / палец)
- `onPointerUp` → заканчиваем drag
- Визуальный thumb и fill — div-ы с pointer-events-none
- Убрать `<input type="range">` и `transform: rotate()` полностью

### Изменение 3: Предотвратить закрытие при тапе на слайдер

Текущий `onMouseLeave` на родителе закрывает панель при уходе пальца за пределы. Нужно:
- Добавить `setPointerCapture(e.pointerId)` при `onPointerDown` чтобы события продолжали приходить
- Или использовать `onPointerLeave` только на внешнем контейнере, не на самом слайдере

## Верификация
1. `npx vite build` — убедиться что билд проходит
2. Открыть в браузере → кликнуть на иконку громкости → потянуть палец вверх-вниз по стеклянной плашке
3. Проверить что громкость меняется плавно от 0 до 100%
4. Проверить что при отпускании пальца панель не закрывается
