# Баг: кнопка "весь экран" до запуска видео

## Корень проблемы

В `GlassPlayer` (AccountCard.jsx) начальное состояние — `started = false`, поэтому рендерится `<img src={poster}>` вместо `<video>`. Элемента `<video>` в DOM нет, `videoRef.current === null`.

Когда пользователь нажимает кнопку fullscreen (Maximize) **до клика по видео**:
1. `toggleFullscreen()` срабатывает → `isFullscreen = true`
2. Fullscreen portal (строка 436) рендерит controls, play icon overlay
3. **Но `<video>` отсутствует** — в container рендерится только `<img>` без стилей `fixed inset-0`
4. Результат: пользователь видит маленькую картинку-превью с oversized контролами, видео не воспроизводится

## Варианты исправления

### Вариант A: Автозапуск видео при входе в fullscreen (рекомендуемый)

В `toggleFullscreen` — если `!started`, вызвать `setStarted(true)` перед toggle.

```jsx
const toggleFullscreen = useCallback(() => {
    if (isTransitioning) return;
    if (!started) setStarted(true);  // ← инициализируем video
    setIsTransitioning(true);
    setDimOpacity(1);
    setDimBlur(40);
    setTimeout(() => {
      setIsFullscreen(prev => !prev);
      setDimOpacity(0);
      setDimBlur(0);
      setTimeout(() => setIsTransitioning(false), 500);
    }, 500);
  }, [isTransitioning, started]);
```

**Плюсы:** минимум изменений, логичный UX (нажал "весь экран" → видео запускается).
**Минусы:** нет паузы между входом в fullscreen и стартом видео (но текущий UX и так подразумевает автостарт при клике).

### Вариант B: Блокировка кнопки fullscreen до старта видео

Просто скрыть/отключить кнопку Maximize, пока `!started`.

```jsx
{started && (
  <button onClick={toggleFullscreen} ...>
    <Maximize size={16} />
  </button>
)}
```

**Плюсы:** ничего не ломается, нет неожиданного поведения.
**Минусы:** пользователь не может сразу посмотреть fullscreen — сначала必須 кликнуть play.

### Вариант C: Рендерить `<video>` всегда (даже до старта), но скрывать

Изменить логику строк 337-356: всегда рендерить `<video>`, а `<img>` использовать только как визуальный оверлей.

```jsx
{(!started && poster) && (
  <img src={poster} alt="" className="absolute inset-0 w-full h-full object-cover z-[1]" />
)}
<video ref={videoRef} src={src} preload="metadata" playsInline
  className={`absolute inset-0 w-full h-full object-cover ... ${!started ? 'opacity-0' : ''}`}
/>
```

**Плюсы:** `<video>` всегда в DOM, fullscreen работает сразу.
**Минусы:** видео загружает metadata даже до клика (минимальный overhead), сложнее поддерживать.

## Рекомендация

**Вариант A** — лучший баланс. Один줄очный фикс, логичный UX, минимальный риск регрессий.

## Файлы для изменения

- `src/components/AccountCard.jsx` — единственное изменение в `toggleFullscreen` (строка ~185)

## Верификация

1. Открыть карточку с видео (не кликая play)
2. Нажать кнопку Maximize → видео должно автоматически стартовать в fullscreen
3. Нажать Maximize снова → выйти из fullscreen
4. Нажать play в обычном режиме → видео стартует как раньше
5. Нажать play, потом Maximize → fullscreen работает как раньше
6. Проверить кнопку "назад" браузера → выходит из fullscreen
